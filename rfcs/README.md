# RunDiff RFC Guide

Visibility: Public

Date: 2026-04-14

## Purpose

RFCs capture larger proposals that are broader than a single implementation decision.

Use an RFC when a proposal changes:

- project scope materially
- supported workflows
- long-term product direction
- major integration surfaces

## Process

1. Draft the RFC from the template.
2. Link the motivation and non-goals clearly.
3. Review the RFC before implementation depends on it.
4. Keep accepted RFCs as decision history even if later changes occur.
